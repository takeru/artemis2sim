# T+0 〜 T+3.4h 合成軌道ドキュメント

## やりたいこと

JPL Horizonsのエフェメリスデータはミッション開始約3.4時間後（T+3.4h）からしか存在しない。
T+0（打ち上げ）からT+3.4hまでの軌道を物理シミュレーションで合成し、
Horizonsデータと滑らかに接続する。

## ミッション概要（T+0 〜 T+3.4h）

この区間で宇宙船は以下を行う:
1. 大気圏内を上昇（打ち上げ）
2. 投入軌道に入る
3. 2回のICPSバーンで高楕円チェックアウト軌道に遷移
4. ICPS分離
5. 近接運用デモ

## 現在の実装

### 処理の流れ（gen_trajectory.py の synthesize_early_trajectory）

```
1. KSC位置計算（ECEF→ICRF、ERA回転）
2. 軌道面・基底ベクトルの決定（Horizonsデータから）
3. 投入軌道パラメータの設定
4. 9パラメータ最適化（scipy Nelder-Mead）
5. 最適化されたパラメータでポイント生成
   Phase 0: 打ち上げ上昇（T+0 → ASCENT_H）
   Phase 1: 投入軌道（ASCENT_H → BURN1_H）
   Phase 2: バーン1後 → バーン2（BURN1_H → BURN2_H）
   Phase 3: バーン2後 → Horizonsデータ開始（BURN2_H → first_met_h）
6. 最終点をHorizonsの最初の点に接続
```

投入軌道は公開値27×2222kmをベースに±15%の微調整を許容。
以前は投入軌道パラメータを完全に自由にしていたため非物理的な値（近地点-200km）に収束していた。
±15%制約により投入軌道が公開値の範囲に留まる。

### 制約（合わせるべき値）

ハード制約（6個）: Horizonsデータの最初の点の状態ベクトル（x, y, z, vx, vy, vz）
ソフト制約（4個）: 投入軌道 ≈ 27×2222km、バーン時刻 ≈ T+49min/T+108min

### コスト関数

2段階最適化:
- Stage 1: 位置合わせ重視 `cost = pos_err + vel_err * 1000`
- Stage 2: 速度方向重視 `cost = pos_err + vel_err * 2000`

Stage 1の結果をStage 2の初期値とする。

### 物理モデル

- 地球点質量重力
- J2（地球扁平率、1.08263×10⁻³）— 3.4hで~400kmの効果
- J3（地球南北非対称性、-2.53×10⁻⁶）— 3.4hで~4km
- 月の重力（3体問題、潮汐力項含む）
- RK4積分、10秒ステップ。Numba @njitでJITコンパイル済み（32倍高速化、2.5秒）
- インパルスバーン（瞬間的な速度変化、バーン時刻は有効中心として最適化）
- Phase 1: burn1からのRK4逆伝播（J2+J3込み、Phase 2+3と同じ物理モデル）
- **RAAN歳差補正:** T+3.4hのHorizons軌道面をT+0に逆補正（-0.39°）。3軌道（投入→中間→チェックアウト）の歳差を個別計算

含まれていないもの:
- J4以降の高次重力場: ~1km以下
- 太陽重力: ~0.04km。無視できる
- 大気抵抗: 投入軌道の近地点27kmを実際には通過しない（バーンが先に実行）。影響なし
- 有限バーン時間: 18分間燃焼をインパルス近似。バーン時刻最適化で有効中心として吸収。dt=1sテストで改善なし確認済み
- 太陽輻射圧、相対論補正、地球の潮汐変形: すべて~0.01km以下

JPL Horizonsの力学モデル（マニュアルより）:
> 「宇宙機の軌道はナビゲーションチームが提供。拡張球面調和関数重力場（数十次以上）、太陽輻射圧、大気抵抗を含む」
我々のJ2+J3モデルとの差が残差の主因。

### 解決済みの問題の経緯

#### 単一シューティング時代

単一シューティング時代は、打ち上げから Horizons 接続点までを 1 組のパラメータで押し通し、
主に終点誤差を最小化する段階。接続誤差はかなり減ったが、公開値 27×2222km との両立が難しく、
遠地点が 2555km まで膨らみやすかった。

##### 接続ジャンプの改善過程
1. 逆伝播（backward RK4）→ 10,000km+
2. Lambert解法 → 7,200km
3. 2バーン射撃法（グリッド探索）→ 913km
4. scipy最適化（5パラメータ）→ 47km / 3.7°
5. 9パラメータ最適化（J2なし）→ 0km / 0°
6. propagate()と出力コードの構造一致 → 0km確認
7. J2有効化+2段階最適化 → 44km / 0.7°
8. RAAN歳差補正 → 30km / 0.0°

##### 投入軌道パラメータの改善過程
1. 完全自由 → 近地点-200km（非物理的）、接続0km
2. 近地点制約 → 27km / 3778km、接続0km
3. 近地点+遠地点制約 → 27km / 2222km、速度3.2°
4. 固定27×2222km → 接続0km、速度4.6°
5. ±15%+J2+2段階 → 31km / 2555km、44km / 0.7°
6. +RAAN補正 → 31km / 2555km、30km / 0.0°

#### multiple shooting 時代

multiple shooting 時代は、burn1後・burn2前・burn2後の中間状態を独立ノードとして持ち、
区間ごとの伝播一致と公開値 27×2222km を同時に解く段階。単一シューティングでは難しかった
「公開値整合」と「T+3.4h 接続」の両立が可能になった。

##### 改善過程
1. finite burn + 月位置時変化 → 公開値整合改善
2. radial抑制 + 最低高度 penalty → 地下通過を解消
3. constrained optimization (SLSQP / COBYLA) 試行 → 可行解探索の限界を確認
4. multiple shooting 導入 → 連結誤差を扱えるように変更
5. continuity 重み調整 → 5km / 12m/s、さらに 3km / 5m/s まで改善
6. burn1 をインパルスとして整理 → 速度不連続を許容する定式化に修正
7. burn1 Δv を nominal 近傍へ拘束 → 非物理な -492m/s 解を解消
8. apo拘束を段階的に強化 → 2555km → 2400km → 2333km → 2289km → 2266km → **2243km**
9. 実質2222km固定（`apo_scale≈1.0`） + multiple shooting → **28km / 2223km、9km / 1.5°（現在）**

→ **multiple shooting によって公開値整合と接続を両立できるようになった。**

#### J2歳差の定量分析

3つの軌道フェーズでのJ2歳差（RAAN + 近地点引数）:

| フェーズ | 軌道 | 時間 | ΔRAAN | Δω |
|---------|------|------|-------|-----|
| 投入軌道 | 27×2222km | 0.82h | -0.18° | +0.29° |
| 中間軌道 | 185×2222km | 0.98h | -0.20° | +0.33° |
| チェックアウト | 2×70355km | 1.6h | -0.01° | +0.02° |
| **合計** | | **3.4h** | **-0.39°** | **+0.64°** |

RAAN補正（-0.39°をh_vecに適用）で位置30km改善。
ω補正は方位角パラメータ(-11.5°)で吸収。

#### 逆伝播アプローチの試行と断念
- Phase 3逆伝播 → 打ち上げ方向-24〜-38°暴走
- ハイブリッド → 改善せず
- 原因: 逆伝播のburn2状態と順方向の投入軌道+バーンの整合性が取れない
- 結論: 全区間順方向+RAAN補正が最も安定

#### 大気抵抗の分析
投入軌道の近地点27kmは成層圏だが、**実際にはバーン前に近地点を通過しない。**
MECOの位置（真近点離角32.8°）から近地点到達までT+108min。バーン1はT+49minで先に実行。
バーンで近地点が185kmに上がるため、27km近地点には到達しない。大気抵抗は無影響。
